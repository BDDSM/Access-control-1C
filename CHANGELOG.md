# История изменений

## 4.6 (в разработке)

### Новое

- **Дерево метаданных конфигурации** — при открытии формируется полностью по всем объектам метаданных: структура и порядок разделов как в конфигураторе (вкладка «Права» роли), иконки типов объектов, подраздел «Общие», вложенность Web/HTTP-сервисов (операции и методы). Фильтры отображения в настройках программы охватывают все типы объектов.
- **Поиск по дереву метаданных** на главной форме: поле над деревом конфигурации, фильтрация по имени и синониму объекта при наборе текста (без Enter). Дерево сужается до подходящих веток и автоматически разворачивается — по тому же принципу, что и поиск роли в поле «Выберите роль».
- **Ограничения доступа** на главной форме: универсальный вывод всех видов и значений ограничений группы доступа; автоматическая активация первой группы и первого вида при выборе роли.
- **Проверка актуальности версии** на главной форме: справа от номера версии — статус («актуальная версия», «доступно обновление», «проверка недоступна»). Сверка с последним релизом на GitHub при открытии формы.
- **Отчёты по ролям** — команда **«Дополнительные функции → Отчеты по ролям»** открывает единую форму с двумя разделами:
  - **Роли без пользователей** — роли, не назначенные ни одному пользователю через группы и профили доступа;
  - **Роли вне профилей** — роли, не входящие ни в один профиль групп доступа.
  - Сводные метрики (найдено / всего ролей / доля от конфигурации), отборы по объектам метаданных и профилям, разворот «Детальный просмотр», поиск по роли, кликабельные метки с расшифровкой списков объектов и профилей.
  - Подсчёт объектов метаданных согласован с главной формой: регистр «Права ролей» + дополнение через `ПравоДоступа` при пустом регистре.

### Исправлено

- Ошибки при фильтрации дерева (некорректные ключи структуры, обращение к удалённым строкам дерева).
- Сброс текста в поле поиска после обновления дерева с сервера.
- Секция «Ограничения доступа» почти всегда была пустой: значения показывались только для видов «Пользователи» и «Организации»; исправлена привязка значений к виду доступа и запись шаблона ограничений.
- Ошибки при листании ролей: недоступный на клиенте конструктор `ХранилищеЗначения`, отсутствующая картинка `ЭлементСправочника` в библиотеке платформы.

### Техническое

- Переработан алгоритм отбора дерева: однопроходная фильтрация вместо полного копирования с последующим удалением строк.
- Debounce 0,3 с перед серверным запросом при наборе текста.
- Ограничения доступа: связь видов и значений по UUID ссылки (`КлючВидаДоступа`), хранение ссылки вида доступа в колонке формы вместо `ValueStorage`.
- Исходники переименованы в `src/AccessControl.v4.6_unlimited/` (ранее `v4.4_unlimited` — имя зафиксировалось при open-source релизе 4.4 и не менялось при сборках 4.5/4.6).
- Версия обработки и регистрационные данные: **v4.6**.
- Скриншоты отчётов: `docs/images/v4.6/role-reports/`.

## 4.5 (2026-05-31)

### Новое

- Поиск роли в поле «Выберите роль» на главной форме: ввод по имени и синониму, фильтрация списка без отдельного поля.
- Сравнение ролей с главной формы: при выборе двух и более ролей в таблице «Роли выбранного объекта» (после выбора объекта в дереве метаданных) открывается форма «Копирование / Сравнение» с автоматическим запуском сравнения; команда «Сравнить роли» снова видна в контекстном меню таблицы.
- Версия обработки отображается как **v4.5**; регистрационные данные дополнительной обработки обновлены.

### Исправлено

- Удалены проверки «пробного периода» (ограничение не применяется к open-source сборке).
- Убрана неработающая заглушка `ОшибкаУжеОбработана()` при разборе журнала регистрации (дедупликация по-прежнему выполняется через `ИсправленныеОшибки` и `ОбъектУжеИзвестен`).
- Константа `УД_ТипОшибокДляУведомленийОбОшибках_tg` читается безопасно: если её нет в конфигурации, используются значения по умолчанию.
- Команда настроек Telegram: понятное сообщение, если отдельная форма уведомлений не подключена в конфигурации.

### Техническое

- Исходники: `src/AccessControl.v4.4_unlimited/` (сборка **v4.5**).
- Удалена отладочная строка в модуле сравнения ролей метаданных.
